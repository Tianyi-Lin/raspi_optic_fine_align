# raspi_optic_fine_align

树莓派上的激光/视觉对准与电机控制项目。

## 监控参数说明

### 主状态栏

多进程运行时，界面顶部会显示一串运行状态，格式大致类似：

```text
多进程运行中 跟踪=ON V=1 C=1 周期ms=20 视觉Hz=29.8 控制Hz=49.9 DetOK=1 Age=0.012s UpdAge=0.015s 水平=12.34 俯仰=-1.20 cmd=... br(...)
```

各字段含义如下：

| 字段 | 含义 | 常见判断方式 |
| --- | --- | --- |
| `跟踪` | 当前跟踪逻辑是否开启 | `ON` 表示控制输出会跟随检测结果，`OFF` 表示只显示不跟踪 |
| `V` | 视觉进程是否存活 | `1` 为正常，`0` 表示视觉进程已退出或异常 |
| `C` | 控制进程是否存活 | `1` 为正常，`0` 表示控制进程已退出或异常 |
| `周期ms` | 控制循环目标周期 | 例如 `20` 表示目标 20ms 一次，理论上约 50Hz |
| `视觉Hz` | 视觉进程实际刷新频率 | 越接近相机/检测设计值越好 |
| `控制Hz` | 控制进程实际循环频率 | 越接近 `1000 / 周期ms` 越好 |
| `DetOK` | 当前检测结果是否有效 | `1` 表示有新鲜有效目标，`0` 表示目标丢失或结果过期 |
| `Age` | 当前用于控制的检测结果“年龄” | 越小越好，过大说明控制拿到的是旧目标 |
| `UpdAge` | 最近一次检测更新时间距现在多久 | 越小越好，和 `Age` 接近时说明检测链路通畅 |
| `水平` | 当前控制进程下发给水平轴的目标角度 | 单位是度 |
| `俯仰` | 当前控制进程下发给俯仰轴的目标角度 | 单位是度 |

### 控制 timing

当打开 `control_debug_timing` 后，主状态栏后面会追加控制耗时信息：

| 字段 | 含义 | 常见判断方式 |
| --- | --- | --- |
| `cmd` | 控制循环里处理命令队列耗时 | 过大说明命令堆积或有阻塞 |
| `set` | 同步设置参数耗时 | 过大说明频繁改配置或 IPC 压力较大 |
| `servo` | 舵机/无刷对象初始化与配置同步耗时 | 正常循环通常应较小，重连时会抬高 |
| `imuI` | IMU 初始化相关耗时 | 正常循环通常接近 0，仅初始化或重连时升高 |
| `imuR` | IMU 读取耗时 | 持续偏大时要检查 IMU 串口链路 |
| `write` | 本轮控制下发电机命令总耗时 | 无刷控制最关键的瓶颈指标之一 |
| `work` | 整个控制循环实际工作耗时 | 若接近 `周期ms`，说明快跑满了 |
| `sleep` | 本轮剩余休眠时间 | 越大说明余量越充足；接近 0 说明控制线程在满负荷跑 |

### 无刷电机 timing `br(...)`

当当前控制模式使用无刷 RS485 电机时，`ctrl_timing` 后面会追加一段 `br(...)`：

```text
br(avg=3.4/pan=1.5/tilt=1.4ms fl=0.00/0.00ms fb=0.0/0.0B io=0.5+0.8/0.5+0.7 skip=12/11 rst=0 par=1.00)
```

各字段含义如下：

| 字段 | 含义 | 常见判断方式 |
| --- | --- | --- |
| `avg` | 单次无刷双轴写入的平均总耗时 | 越小越好，是无刷链路总体开销 |
| `pan` | 水平轴单次命令平均耗时 | 用于判断水平轴是否更慢 |
| `tilt` | 俯仰轴单次命令平均耗时 | 用于判断俯仰轴是否更慢 |
| `fl` | flush 平均耗时，格式为 `pan/tilt` | 持续偏大说明串口前置清理本身在耗时 |
| `fb` | flush 平均丢弃字节数，格式为 `pan/tilt` | 持续非零说明输入缓冲中经常有残留脏数据 |
| `io` | 发送耗时 + 等待回复耗时，格式为 `pan_send+pan_reply / tilt_send+tilt_reply` | 回复时间偏大通常比发送时间更值得关注 |
| `skip` | 因目标角度和速度未变化而跳过发送的次数，格式为 `pan/tilt` | 持续增长说明“重复命令抑制”在起作用 |
| `rst` | 本统计周期内电机对象被重置/重连的次数 | 非零通常意味着串口异常、回复异常或校验失败 |
| `par` | 并行写比例 | `1.00` 表示统计周期内基本都在并行写，`0.00` 表示串行写 |

### IMU 状态栏

界面里 IMU 状态通常会显示当前姿态和归零基准：

| 字段 | 含义 | 常见判断方式 |
| --- | --- | --- |
| `Pitch` | 当前 IMU 俯仰角 | 单位是度 |
| `Yaw` | 当前 IMU 航向角 | 单位是度 |
| `Age` | 最近一次 IMU 更新距现在多久 | 越小越好，过大说明 IMU 数据在变旧 |
| `Pitch Base` | 当前 IMU 归零基准俯仰角 | 执行零点校准后更新 |
| `Yaw Base` | 当前 IMU 归零基准航向角 | 执行零点校准后更新 |
| `Pitch Delta` | 当前俯仰角相对归零基准的偏差 | 自动稳定时会用到 |
| `Yaw Delta` | 当前航向角相对归零基准的偏差 | 自动稳定时会用到 |

## 看数值时的几个经验

| 现象 | 可能含义 |
| --- | --- |
| `sleep` 很小、`work` 接近 `周期ms` | 控制线程接近跑满 |
| `write` 大、`io` 里的 reply 明显偏大 | 电机回复等待是主要瓶颈 |
| `fb` 经常非零 | 串口输入中常有旧包或脏数据残留 |
| `rst` 偶尔出现 | 单次异常可接受，但持续增长要排查 RS485 时序/线材/供电 |
| `DetOK=0` 且 `Age=-1` | 当前没有可用于控制的有效检测结果 |
